# News Type Playbook

## 选择流程

1. 判断时效压力：是否需要分钟级或小时级更新。
2. 判断叙事中心：是“事件进展”、还是“议题解释”、还是“人物观点”。
3. 判断证据深度：是否需要多源交叉、长链条解释、反证与争议处理。
4. 选定新闻类型并调用对应模板；如混合型任务，先选主类型，再附加次类型元素。

## 自动判定机制（按新闻话题触发）

当用户仅提供“新闻话题”且未指定类型时，必须自动判定，不得直接默认。

### A. 关键词触发（第一优先）

- 命中 `突发/刚刚/最新进展/持续更新/通报中` -> `实时新闻`
- 命中 `专访/对话/采访/问答/访谈` 且主对象为个人 -> `人物专访`
- 命中 `专题/多方观点/圆桌/行业怎么看` -> `专题采访`
- 命中 `原因/影响/争议/拆解/深挖/为何` -> `深度报道`
- 命中 `发布/通报/快讯/简讯/简报/公告` -> `简报`

### B. 评分判定（关键词冲突时启用）

按题目与任务描述打分，取最高分作为主类型：

| 信号 | 简报 | 深度报道 | 专题采访 | 实时新闻 | 人物专访 |
|---|---:|---:|---:|---:|---:|
| 时效性（分钟/小时级） | 1 | 0 | 0 | 3 | 0 |
| 需要解释复杂因果 | 0 | 3 | 1 | 0 | 1 |
| 需要多方受访者观点 | 0 | 1 | 3 | 0 | 1 |
| 核心主体是单一人物 | 0 | 1 | 1 | 0 | 3 |
| 任务强调“快/短/立即发布” | 3 | 0 | 0 | 2 | 0 |

平分处理：
1. 若含强时效词，优先 `实时新闻`。  
2. 若含“专访/人物”词，优先 `人物专访`。  
3. 仍平分时，优先 `深度报道`（信息价值更完整）。

### C. 输出要求

正文前必须先给出 `类型判定卡`（3行以内）：
- `判定结果`：主类型（可附次类型）
- `触发依据`：关键词或评分信号
- `采用模板`：模板文件路径

## 发布态排版规则（面向读者）

- 正文采用“连续叙事”为主，不堆叠编辑型小标题。
- 小标题仅用于段落转折，不用于罗列检查项。
- 各类型小标题上限：
  - `简报`：0个
  - `实时新闻`：最多1个
  - `深度报道`：最多2个
  - `专题采访`：最多2个
  - `人物专访`：最多2个
- 采编信息（核验、方法、待确认）统一放到正文后的附录区域。
- 图文排版遵循：导语后首图，段落证据点后插图，每图带出处。

## 各类型写作方法

### 1) 简报

- 目标：最快交付可核验事实与影响结论。
- 结构：导语一句话 -> 连续正文2-4段 -> 收束句。
- 必写：时间、地点、主体、动作、影响对象、关键数字口径。
- 核验重点：数字与时间口径一致；标题不超事实边界。

### 2) 深度报道

- 目标：解释“为什么发生、影响什么、未来如何变化”。
- 结构：问题提出 -> 证据链（数据/政策/行业）-> 分歧与反证 -> 结论（最多2个小标题）。
- 必写：核心问题、证据来源、反方观点、方法边界。
- 核验重点：每个关键结论至少双来源；结论与证据链一一对应。

### 3) 专题采访

- 目标：围绕同一议题呈现多方观点与可执行信息。
- 结构：议题背景 -> 连续问答叙事 -> 观点对照 -> 记者总结（最多2个小标题）。
- 必写：提问逻辑、答复原意、删改说明（如有）、交叉核验结果。
- 核验重点：引述准确，避免断章取义；每条关键主张补充外部证据。

### 4) 实时新闻

- 目标：在信息不完整时，稳定输出“已确认事实”并持续更新。
- 结构：最新进展 -> 时间轴 -> 服务信息（正文小标题最多1个）。
- 必写：每条更新时间戳、版本号、来源状态（已核验/待核验）。
- 核验重点：不把传言写成结论；每次更新标注新增与更正。

### 5) 人物专访

- 目标：展示人物观点、决策依据与公共影响。
- 结构：人物背景 -> 核心观点与问答 -> 争议回应 -> 记者观察（最多2个小标题）。
- 必写：人物身份与利益关系、观点证据、可验证事实。
- 核验重点：人物自述与外部事实分层呈现；避免宣传稿口吻。

## 类型冲突处理

- 事件突发但要解释背景：主类型 `实时新闻`，次元素引用 `深度报道` 的背景段。
- 单一采访但信息密集：主类型 `专题采访`，必要时补 `深度报道` 的证据链段落。
- 人物发言引发市场波动：主类型 `人物专访`，首段按 `简报` 写出可验证结果。
